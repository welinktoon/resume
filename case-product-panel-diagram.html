Диаграмма 02

Жизненный цикл сигнала риска

Эта схема показывает, что происходит после обновления данных и как сигнал превращается в действие команды.

1 Данные обновились

панель получила задачи, встречи и статусы источников

2 Правило сработало

найдена просрочка, сбой источника или задача без движения

3 Сигнал показан

риск виден в дашборде, фильтре или карточке задачи

4 Команда действует

появляется вопрос, созвон, эскалация или обновление статуса

5 Риск закрыт

задача обновлена, источник восстановлен, блокер снят

Диаграмма 04

API-методы и контракты данных

Минимальный набор ручек для MVP: получение данных, запуск синхронизации и работа с сигналами риска.

GET /api/tasks

Параметры: status, assignee, riskOnly, updatedBefore. Ответ: список задач для доски и фильтров.

GET /api/calendar/events

Параметры: dateFrom, dateTo, owner. Ответ: встречи недели, ближайшее событие и плотность дня.

GET /api/connections

Параметры: source, status. Ответ: состояние источников, время синхронизации и текст ошибки.

POST /api/sync/run

Тело: source, force. Ответ: syncId, status, startedAt. Запускает ручное обновление источника.

GET /api/risks/signals

Параметры: severity, type, entityType. Ответ: активные сигналы по задачам, встречам и источникам.

PATCH /api/signals/{id}

Тело: status, comment. Ответ: обновленный сигнал. Нужен для отметки "взято в работу" или "закрыто".

Yandex Tracker REST POST /v3/issues/_search

получение задач по фильтру: очередь, статус, исполнитель, дата обновления

Google Calendar API events.list

получение встреч за период: timeMin, timeMax, calendarId, singleEvents

Google Sheets API spreadsheets.values.get

чтение рабочих списков, партнеров, сверок и справочников

Gmail API users.messages.list

проверка входящих коммуникаций и приглашений по рабочим фильтрам

Успешный ответ

{ data, meta: { source, syncedAt, total } }

Ошибка источника

{ errorCode, errorText, source, retryAvailable }

Общее правило

панель показывает кэш и время последней синхронизации, если источник временно недоступен

экономия времени -40 мин/день

на ручной сбор статусов из трекера, календаря, таблиц и почты

меньше переключений 4 сервиса → 1 экран

контекст задачи, встречи и подключения видны рядом

раньше видны блокеры риски до созвона

команда приходит на статус уже с понятным списком проблем
